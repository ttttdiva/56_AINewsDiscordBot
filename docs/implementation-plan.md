# 実装計画書

更新日: 2026-03-11

## 目的

Discord の指定チャンネルに、毎日 1 回だけ AI 関連ニュースの要約を投稿する Bot を作る。

## 採用方針

- Python ツール向けのシンプルなドキュメント構成を使う
- X 検索は MCP ではなく Grok API 直呼びにする
- Discord 部分は `discord.py` の標準的な設定導線と起動方法に寄せる
- 先に MVP を閉じ、その後に拡張する

## MVP 要件

- 1 日 1 回、設定時刻に実行される
- Grok `x_search` で AI 関連の投稿を収集する
- ノイズを落として 3-7 件程度のトピックを要約する
- Discord の 1 チャンネルに Markdown 形式で投稿する
- 過去に送った投稿 URL を重複送信しない
- 失敗時はログに原因が残る

## フェーズ

### Phase 0: リポジトリ初期化

- Python 実行環境を決める
- `pyproject.toml` を追加する
- `.env.sample` と `.gitignore` を追加する
- ログ出力方針を決める

完了条件:
- `python -m` か単一エントリポイントで起動できる骨組みがある

### Phase 1: 設定とモデル定義

- 環境変数を読み込む設定クラスを作る
- 検索条件、投稿先、実行時刻、タイムゾーンを設定化する
- 収集結果、要約結果、送信履歴のデータモデルを作る

完了条件:
- `.env` だけで Bot の基本挙動を切り替えられる

### Phase 2: Grok API クライアント

- `x_search` を使う Grok API 呼び出しを Python に実装する
- タイムアウト、HTTP エラー、レスポンス抽出を実装する
- 検索プロンプトを Bot 用に固定化する

完了条件:
- 手動実行で AI 関連投稿の取得結果をローカルに保存できる

### Phase 3: 収集、フィルタ、要約

- 検索キーワード群を定義する
- 収集結果を正規化する
- URL 単位または投稿 ID 単位で重複排除する
- 投稿候補をランキングして日次要約を組み立てる

完了条件:
- 1 回の実行で Discord 投稿用 Markdown を生成できる

### Phase 4: 状態保存

- SQLite で実行履歴と送信済み投稿を保存する
- 同一日の再実行時に重複送信を防ぐ
- 障害時の再送に備えて投稿本文も保存する

完了条件:
- 再実行しても同じ投稿を二重送信しない

### Phase 5: Discord 投稿

- `discord.py` で Bot を起動する
- 指定チャンネルへメッセージ送信する
- 2000 文字超過時は分割送信する
- 起動時の疎通確認ログを出す

完了条件:
- 手動実行で Discord へ正常投稿できる

### Phase 6: スケジューラ

- JST 基準で 1 日 1 回のジョブを登録する
- 起動中の二重実行を防ぐロックを入れる
- 手動即時実行モードも持たせる

完了条件:
- 常駐実行で毎日決まった時刻に 1 回だけ動く

### Phase 7: テストと運用手順

- 単体テストを追加する
- API モックで要約生成まで検証する
- Discord 設定手順と `.env` 設定例を `docs/` に追加する

完了条件:
- 初回セットアップから日次運用まで README だけで追える

## 初回実装で作る想定ディレクトリ

- `src/config/`
- `src/clients/`
- `src/services/`
- `src/storage/`
- `src/bot/`
- `src/scheduler/`
- `tests/`

## 主要リスク

- Grok `x_search` の返却形式が揺れる可能性
- X 上の話題はノイズが多く、単純検索では宣伝投稿が混ざる
- Discord 投稿文が 2000 文字を超えやすい
- 毎日 1 回でも、失敗時の再実行方針を決めないと運用が荒れる

## 実装開始時の最初の 3 タスク

1. `pyproject.toml` と `.env.sample` を作る
2. `src/config/settings.py` と `src/clients/grok_client.py` を作る
3. 手動実行で検索結果を JSON 保存する CLI を作る
